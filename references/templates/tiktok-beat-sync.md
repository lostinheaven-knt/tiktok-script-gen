# TikTok 场景模板：卡点踩拍 / 节奏同步

> 知识截止：2026-06
> 场景 ID: tiktok-beat-sync
> 分类：TikTok 专属 · 节奏型快剪
> source: technical-design.md §5.4

---

## 1. 场景定义

适用于舞蹈、挑战、快剪带货、产品多角度展示等需要与 BGM 节拍同步的视频段落。通过在节拍点上切镜，制造视觉与听觉的同步快感，大幅提升完播率和重播率——这是 TikTok 生态中 RPM 最高的内容形态之一。

**核心设计原则**：切镜节奏由 BPM 驱动而非随意排列；单 prompt 最多 3 镜头；每个镜头 ≥0.4 秒（Seedance 物理合理性约束，过短易模糊）。

---

## 2. 关键变量列表

| 变量 | 说明 | 示例值 |
|---|---|---|
| `{bpm}` | 音乐 BPM | 128 |
| `{beat_interval_seconds}` | 每拍时长（60 / bpm） | 0.47s（128 BPM） |
| `{cut_interval}` | 切镜间隔（按拍数） | 每 2 拍（0.94s）/ 每 4 拍（1.88s） |
| `{shot_count}` | 镜头数量 | 2-3（单 prompt 上限 3） |
| `{product}` | 产品描述 | "运动鞋" / "无线耳机" / "唇釉套组" |
| `{angles}` | 拍摄角度列表 | 正面 / 45 度俯拍 / 侧面 / 底部翻转 |
| `{lighting_style}` | 灯光风格 | "环形柔光" / "霓虹脉冲光" / "高对比硬光" |
| `{background}` | 背景类型 | "纯黑背景" / "白色无缝背景" / "渐变霓虹" |
| `{sync_type}` | 同步类型 | action-on-beat（动作发生在节拍点）/ cut-on-beat（切镜发生在节拍点） |

---

## 3. 推荐时长

| 时长 | BPM 覆盖 | 典型镜头数 |
|---|---|---|
| **8-12 秒** | 16-24 拍（128 BPM）| 2-3 镜头 |
| **12-16 秒** | 24-32 拍（128 BPM）| 需拆为 2 个 segment（每个 ≤3 镜头） |
| **16-20 秒** | 32-40 拍（128 BPM）| 需拆为 2-3 个 segment |

> 超过 20 秒的视频建议拆分为多个 beat-sync segment，之间用 continuity_hint 衔接过渡。

---

## 4. 典型 shot_type

| 镜头类型 | 说明 | 典型切镜间隔 |
|---|---|---|
| **rapid-cut** | 快速切镜，多角度多构图 | 每 2-4 拍切一次 |
| **multi-angle** | 同一产品/主体多角度轮换 | 每 4 拍切一次 |
| **action-sync** | 动作发生在节拍点上 | 动作 = 节拍点，切镜 = 每 8 拍 |

---

## 5. BPM 与镜头数对应规则

| BPM 范围 | 单拍时长 | 推荐切镜间隔 | 8 秒内最多镜头数 | 20 秒内最多镜头数 |
|---|---|---|---|---|
| 70-90（慢速） | 0.67-0.86s | 每 2 拍（1.3-1.7s） | ~5 镜头 | ~12 镜头（需拆 3+ segment） |
| 90-110（中速） | 0.55-0.67s | 每 2-4 拍（1.1-2.7s） | ~4 镜头 | ~10 镜头（需拆 3+ segment） |
| 110-130（快速） | 0.46-0.55s | 每 4 拍（1.8-2.2s） | ~3 镜头 | ~8 镜头（需拆 2+ segment） |
| 130-150（极快） | 0.40-0.46s | 每 4-8 拍（1.6-3.7s） | ~2-3 镜头 | ~6 镜头（需拆 2+ segment） |

> **核心约束**：无论 BPM 多快，单镜头时长不得短于 **0.4 秒**（Seedance 物理合理性约束）。超过 130 BPM 时优先减少镜头数而非压缩单镜头时长。

## 6. Seedance prompt 示例

### 产品快剪（product-rotation 映射）— 128 BPM，112 字

```
镜头1：运动鞋正面特写 0.5 秒，纯黑背景。
镜头2：运动鞋 45 度俯拍 0.5 秒，鞋底纹路清晰。
镜头3：运动鞋侧面全景 1 秒，轻微环绕运镜展示鞋身流线。
整体节奏匹配 128 BPM 音乐节拍点，每 2 拍切镜一次。
```

### 多角度产品（orbit-shot 映射）— 120 BPM，108 字

```
镜头1：无人机正面悬浮特写 0.8 秒，白色无缝背景，环形柔光。
镜头2：无人机俯拍 0.8 秒，展示旋翼展开状态。
镜头3：无人机底部翻转视角 1.2 秒，LED 尾灯脉冲闪烁。
整体节奏匹配 120 BPM，每 4 拍切镜一次。运镜为固定机位。
```

### 舞蹈快剪（product-rotation 映射）— 140 BPM，105 字

```
镜头1：模特全景正面舞步 1 秒，霓虹紫背景。
镜头2：模特侧面近景手臂动作 1 秒，霓虹蓝背景。
镜头3：模特俯拍旋转收尾 1.5 秒，霓虹渐变背景。
节奏匹配 140 BPM，每 8 拍切换。三镜头均为固定机位。
```

## 7. 典型失败反例

### 失败 prompt

```
超快速卡点显示，8K 超高清，完美踩拍，
产品在 0.2 秒内切换 8 个角度，眼花缭乱的绝美展示
```

**失败原因**：
- "超快速"、"8K"、"超高清"、"完美"、"绝美" → 违反 R1
- 0.2 秒单镜头 → 违反 0.4 秒最小时长约束
- 8 个角度 → 超过单 prompt 3 镜头上限
- BPM 和具体节拍间隔未描述 → 音乐无法匹配

## 9. music_suggestion 特殊要求

此模板**必须**输出 `music_suggestion.bpm_range`，且与 prompt 内描述的节拍数一致。否则下游无法将音频与视频同步。

| music_suggestion 字段 | 要求 |
|---|---|
| `bpm_range` | **必填**，且与 prompt 中的切镜间隔计算一致 |
| `mood` | 必填（如 "upbeat energetic" / "synth wave"） |
| `tiktok_library_keywords` | ≥1 条关键词（AC9） |

**示例**：
```yaml
music_suggestion:
  mood: "upbeat electronic pop"
  bpm_range: "126-130"
  license_safe: true
  tiktok_library_keywords:
    - "trending tiktok sound electronic beat"
    - "commercial library upbeat pop 128bpm"
```

## 11. 注意事项（Caveats）

1. **必须输出 `music_suggestion.bpm_range`**，且与 prompt 内描述的节拍数一致。这是下游音视频同步的唯一依据。
2. **Seedance 单 prompt 最多 3 镜头**：超过 3 个切镜必须拆成多个 scene/segment。如需展示 5 个角度，拆为 segment A（3 镜头）+ segment B（2 镜头），用 continuity_hint 衔接。
3. **单镜头时长不短于 0.4 秒**：Seedance 物理合理性约束——过短切镜会导致画面模糊（模型需要至少 0.4s 来建立帧间连贯）。超快 BPM（>140）优先减少镜头数。
4. **卡点点位用"约 X 秒"描述**：不写"完美卡点"（违反 R1），改用"匹配 128 BPM 节拍点"、"每 4 拍切镜一次"等可量化描述。
5. **三镜头机位避免全同一角度**：至少包含 2 个不同角度（正面/俯拍/侧面），否则快剪的"变化感"不足。
